# Cursor Agent Preset

This preset is optimized for working with Cursor Agent mode, focusing on the core actions you need when reviewing and applying AI-generated changes.

## Key Mappings

| Key | Icon | Action | Shortcut |
|-----|------|--------|----------|
| **Key 1** | вң… | Apply change | `Enter` |
| **Key 2** | вҸӯ | Apply and add to Allowlist | `Shift + Enter` |
| **Key 3** | рҹҷ…вҖҚвҷӮпёҸ | Skip | `ESC` |
| **Key 4** | вқҢ | Cancel operation | `Ctrl + C` |
| **Key 5** | рҹ’ӯ | Agent вҶ’ Plan вҶ’ Ask | `Win + .` |
| **Key 6** | рҹҺӨ | Start dictation | `Shift + Win + Space` |

## Detailed Configuration

### Key 1: Apply (вң…)
**Shortcut:** `Enter`

Accept and apply the current suggested change. This is your primary action when Cursor Agent proposes a modification you agree with.

```
Key: Enter
Action: Apply suggested change
```

### Key 2: Apply and Allowlist (вҸӯ)
**Shortcut:** `Shift + Enter`

Apply the change and add this type of modification to the allowlist. Future similar changes will be applied automatically without asking for confirmation.

```
Key: Shift + Enter
Action: Apply and trust this change type
```

### Key 3: Skip (рҹҷ…вҖҚвҷӮпёҸ)
**Shortcut:** `ESC`

Skip the current suggestion without applying it. Use when the proposed change is not what you want.

```
Key: ESC
Action: Skip current suggestion
```

### Key 4: Cancel (вқҢ)
**Shortcut:** `Ctrl + C`

Cancel the current operation. Use when you want to stop the agent mid-generation or abort an ongoing task.

```
Key: Ctrl + C
Action: Cancel current operation
```

### Key 5: Mode Toggle (рҹ’ӯ)
**Shortcut:** `Win + .`

Cycle through Cursor modes: Agent вҶ’ Plan вҶ’ Ask. Quickly switch between different AI interaction styles.

```
Key: Win + .
Action: Toggle Agent/Plan/Ask mode
```

### Key 6: Dictation (рҹҺӨ)
**Shortcut:** `Shift + Win + Space`

Start voice dictation for hands-free input. Great for explaining complex requirements or describing bugs.

```
Key: Shift + Win + Space
Action: Start voice input
```

## Usage Patterns

### Quick Review Flow

When Cursor Agent suggests changes:

1. Review the proposed change
2. **вң…** Apply if it looks good
3. **вҸӯ** Apply and allowlist for repetitive trusted changes
4. **рҹҷ…вҖҚвҷӮпёҸ** Skip if you want to handle it differently
5. **вқҢ** Cancel if the agent is going in the wrong direction

### Building Trust with Allowlist

Use **вҸӯ** (Apply and Allowlist) for:
- Formatting changes you always accept
- Import organization
- Type annotations
- Code style fixes

Once allowlisted, these changes apply automatically, speeding up your workflow.

### Mode Switching

Use **рҹ’ӯ** to quickly switch between:
- **Agent** вҖ” Full autonomous mode
- **Plan** вҖ” See the plan before execution
- **Ask** вҖ” Simple Q&A mode

### Voice-Driven Development

1. Press **рҹҺӨ** to start dictation
2. Describe what you want to build or fix
3. Let the agent generate the solution
4. Review and apply with **вң…** or **вҸӯ**

## Why These Icons?

- **вң…** (Check mark) вҖ” Approve and apply
- **вҸӯ** (Next track) вҖ” Apply and move forward automatically
- **рҹҷ…вҖҚвҷӮпёҸ** (Person gesturing NO) вҖ” Skip past this one
- **вқҢ** (Cross mark) вҖ” Stop/cancel
- **рҹ’ӯ** (Thought bubble) вҖ” Switch thinking mode
- **рҹҺӨ** (Microphone) вҖ” Voice input

## Pro Tips

### Batch Similar Changes
When Cursor suggests multiple similar changes (like adding types to function parameters), use **вҸӯ** on the first one. The rest will apply automatically.

### Review Before Allowlist
Only use **вҸӯ** when you are confident about the change pattern. Once allowlisted, similar changes skip the review step entirely.

### Quick Rejection
**рҹҷ…вҖҚвҷӮпёҸ** (Skip) is faster than clicking with the mouse. Keep your hands on the keypad for maximum flow.

### Voice for Context
Use **рҹҺӨ** when you need to explain:
- Complex refactoring requirements
- Bug descriptions with context
- Feature specifications
- Code review feedback

Voice is much faster than typing for detailed explanations.
